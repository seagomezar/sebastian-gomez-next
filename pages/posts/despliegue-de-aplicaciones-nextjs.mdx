# Despliegue de aplicaciones Next.js con vercel

<PostHeader
  tags={["Next.js", "Javascript", "NodeJS", "Desarrollo Web"]}
  date={1615218289470}
  slug="despliegue-de-aplicaciones-nextjs-con-vercel"
/>

Dependiendo de  como has contruido tu aplicación podras desplegarla 
a una u otra parte. Por ejemplo si tu aplicación esta completamente
generada como estática SSG, de esto hablamos en este post: [Modos de renderizado en Next.js](/posts/modos-de-renderizado-en-nextjs)
puede usar un servidor básico que te permita mostrar archivos estáticos. Por ejemplo Firebase Hosting o Github Pages.

Para ellos basta con ejecutar el comando:

```bash
next export
```

Y subir la carpeta donde corresponda o construir tu propio sistema de integración continua y pipelina
de despliegue.

⚠️**Tip:** Ten presente que si ese es tu camino deberás reescribir las rutas a peticiones 
que usen variables de entorno de testing, development o producción.

Sin embargo hay una manera mucho más sencilla. Vercel es la compañía detras de Next.js
ellos literalmente crearon Next.js y además tienen un servicio gratuito para aplicaciones
construidas con Next.js donde es solo usar un comando y desplegar tu aplicación.

Familiarizate con el sitio web de [Vercel](https://vercel.com), es importante ya que
allí podrás descubrir funcionalidades interesantes. Además entenderás como funciona la carpeta
gratuita (Que para ser gratis es muy muy grande).

Primero instalemos Vercel como paquete global de Node:

```bash
npm i -g vercel
```

Luego simplemente debes ejecutar el comando:

```bash
$ vercel
````

En tu consola y deberá seguir los pasos.

También el sitio web puedes configurarlo desde tu repositorio de Github
y hacer todo el enlace entre ambos sitios:
